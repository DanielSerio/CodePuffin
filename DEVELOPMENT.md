## Rules

### General

- Always represent the truth.
- Always prefer simple solutions over clever ones.
- Always prefer readable code over concise code.
- Always prefer maintainable code over optimized code.
- Always use SOLID principles.
- Always use DRY principle.
- Always use KISS principle.
- Always use YAGNI principle.

### TypeScript

- Never use `any` type.
- Prefer inference over explicit types.
- Prefer utility types over manual type definitions.
- Comment the code so that humans can understand it.
